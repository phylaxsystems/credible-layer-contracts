**FILE 3: test/integration/StateOracleOwnerOnChain.t.sol**

Concrete implementation: AdminVerifierOwner + DAVerifierOnChain.

- `_deployDAVerifier`: return `new DAVerifierOnChain()`
- `_registerAdopter`: same as Owner+ECDSA (deploy OwnableAdopter)
- `_generateValidProof`: return `(new bytes(0), proof)` where `proof` is arbitrary bytecode and the assertionId used must equal `keccak256(proof)`. Override `test_addAssertionWithValidProof` or use a helper that generates `(assertionId, proof)` pairs.

IMPORTANT: For DAVerifierOnChain, the assertionId must equal `keccak256(proof)`. The base class passes arbitrary assertionId to `_generateValidProof(assertionId)`. The OnChain concrete contracts must handle this by:
- For fuzz tests: compute `assertionId = keccak256(proof)` where `proof` is some deterministic bytes derived from the fuzz input
- OR: override `_generateValidProof` to return proof such that `keccak256(proof) == assertionId`. This means `proof` must be fixed and `assertionId` must be pre-computed.

**Recommended approach**: The base class should define `_generateAssertionId(bytes memory proof)` that concrete contracts override. For ECDSA, assertionId is arbitrary (signature validates any ID). For OnChain, assertionId must be `keccak256(proof)`.

Actually, simpler: make the fuzz test in the base class pass a `bytes32 seed` and let concrete contracts generate `(assertionId, metadata, proof)` from the seed:
- `function _generateValidAssertion(bytes32 seed) internal virtual returns (bytes32 assertionId, bytes memory metadata, bytes memory proof);`
- For ECDSA: `assertionId = seed`, sign it
- For OnChain: `proof = abi.encode(seed)`, `assertionId = keccak256(proof)`

And `_generateInvalidAssertion(bytes32 seed)` similarly.
